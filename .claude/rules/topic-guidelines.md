# トピック選定ガイドライン

このZennアカウントで執筆する記事のトピック選定に関するガイドラインです。

---

## このアカウントの主要テーマ

以下の3テーマを軸に記事を執筆します。トピック選定時はこれらに沿っているか確認してください。

### 1. AI導入・活用

```
✅ 推奨トピック
- Claude Code、GitHub Copilot等のAIツール導入記事
- チームへのAIツール定着・浸透施策
- AI活用の効果測定・ROI分析
- プロンプトエンジニアリングの実践
- AIコードレビューの活用

❌ 避けるべき切り口
- 「AIは仕事を奪う」等の煽り記事
- 特定AIサービスの批判記事
- 未検証のAI能力に関する主張
```

### 2. 生産性向上

```
✅ 推奨トピック
- 開発チームの生産性計測手法
- ベロシティ・リードタイム等のメトリクス設計
- 開発プロセス改善の実践記録
- 時間短縮・効率化の具体的手法
- ツール導入による改善効果の定量化

❌ 避けるべき切り口
- 「残業は悪」等の働き方論争
- 特定の開発手法の全否定
- 定量的データのない主張
```

### 3. 開発体制改善

```
✅ 推奨トピック
- devcontainer、CI/CD等の開発環境整備
- コードレビュープロセスの改善
- ドキュメント駆動開発の実践
- チーム開発のベストプラクティス
- オンボーディング改善

❌ 避けるべき切り口
- 他社の開発体制批判
- 「〇〇は時代遅れ」等の断定
- 理想論だけで実践を伴わない内容
```

## 炎上リスクの高いトピック

以下のトピックは炎上リスクが高いため、**原則として避ける**か、**慎重に執筆**してください。

### 🔴 原則避けるべきトピック

| トピック | リスク | 理由 |
|----------|--------|------|
| プログラミング言語の優劣比較 | 高 | ファン同士の対立を招く |
| フレームワークの優劣比較 | 高 | 特定コミュニティからの批判 |
| エディタ・IDEの優劣比較 | 高 | 宗教戦争になりやすい |
| 特定企業・サービスの批判 | 高 | 法的リスク・炎上リスク |
| キャリア論・年収論 | 高 | 価値観の対立を招く |
| 採用・面接の内情暴露 | 高 | 機密情報漏洩リスク |

### 🟡 慎重に執筆すべきトピック

| トピック | 注意点 |
|----------|--------|
| 開発手法の比較（アジャイル vs ウォーターフォール等） | 優劣ではなく「状況による」という姿勢で |
| 失敗談・障害報告 | 顧客対応・改善策を必ず含める |
| 退職エントリ | 前職の批判にならないよう注意 |
| 技術選定の理由 | 選ばなかった技術の否定にならないよう注意 |

## 良いトピック選定のチェックリスト

新しい記事を書く前に、以下を確認してください。

### 1. テーマ適合性

- [ ] 3つの主要テーマ（AI、生産性、開発体制）のいずれかに該当する
- [ ] または、それらに関連する周辺トピックである

### 2. 実践ベース

- [ ] 実際に自分で経験・検証した内容である
- [ ] 理論だけでなく、具体的な実践例がある
- [ ] 「〜らしい」「〜だそうだ」ではなく、自分の言葉で書ける

### 3. 定量的要素

- [ ] 可能な限り数値・データを含められる
- [ ] Before/After の比較ができる
- [ ] 効果測定の結果を示せる

### 4. 再現可能性

- [ ] 読者が同じ手順で追試できる
- [ ] 必要な環境・ツールのバージョンが明記できる
- [ ] 前提条件が明確である

### 5. 炎上リスク

- [ ] 炎上リスクの高いトピックに該当しない
- [ ] 断定的な主張を避けられる
- [ ] 特定の技術・企業の批判にならない
