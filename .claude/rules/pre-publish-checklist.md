# 公開前チェックリストルール

記事を `published: true` に変更する前に、必ず以下のチェックを完了すること。

---

## 必須チェック（公開をブロック）

以下がすべて完了していない場合、公開を中止してください。

### 1. 炎上リスクチェック

```
/check-controversy を実行し、🔴 高リスクがゼロであること
```

- [ ] 顧客不在のHard Things自慢がない
- [ ] マサカリ誘発表現（断定・広い主語）がない
- [ ] 機密情報・個人情報が含まれていない
- [ ] 上から目線・マウンティング表現がない
- [ ] 誤情報・未検証情報がない

### 2. 技術的正確性

- [ ] コードは実際に動作確認済み
- [ ] バージョン情報が明記されている
- [ ] 再現可能な手順になっている

### 3. Front Matter

- [ ] title: 設定済み（60文字以内推奨）
- [ ] emoji: 内容に合った絵文字
- [ ] type: tech または idea
- [ ] topics: 1〜5個の関連タグ
- [ ] published: false → true に変更予定

---

## 推奨チェック（品質向上）

以下は推奨項目です。可能な限り対応してください。

### 4. 記事レビュー

```
/review-article を実行し、総合評価 ⭐⭐⭐⭐☆ 以上であること
```

- [ ] 読者視点: 対象者・メリットが明確
- [ ] 構成: 論理的な流れ、適切な見出し階層
- [ ] 引用: 出典が明記されている
- [ ] SEO: タイトル・topicsが適切

### 5. 校正チェック

```
/proofread を実行し、重大な誤りがないこと
```

- [ ] 誤字脱字がない
- [ ] 表記ゆれがない
- [ ] 文体が統一されている

### 6. プレビュー確認

```
npx zenn-cli preview でプレビュー確認
```

- [ ] レイアウトが崩れていない
- [ ] 画像が正しく表示されている
- [ ] コードブロックのハイライトが正しい
- [ ] リンクが有効である

## 公開後の確認

- [ ] Zennに正しくデプロイされている
- [ ] OGP画像が正しく生成されている
- [ ] 本番URLでアクセスできる

---

## 緊急時の対応

公開後に問題が発見された場合：

1. **軽微な修正**: 即座に修正してpush
2. **重大な問題**: `published: false` に戻して非公開化
3. **炎上発生時**: 冷静に対応、必要に応じて謝罪・訂正

```bash
# 緊急非公開化
sed -i '' 's/published: true/published: false/' articles/対象記事.md
git add .
git commit -m "緊急非公開化: 〇〇の理由"
git push
```
