# 公開前チェックリストルール

記事を `published: true` に変更する前に、必ず以下のチェックを完了すること。

---

## 必須チェック（公開をブロック）

以下がすべて完了していない場合、公開を中止してください。

### 1. 炎上リスクチェック

```
/check-controversy を実行し、🔴 高リスクがゼロであること
```

- [ ] 顧客不在のHard Things自慢がない
- [ ] マサカリ誘発表現（断定・広い主語）がない
- [ ] 機密情報・個人情報が含まれていない
- [ ] 個人・所属会社を特定できる情報がない
- [ ] 上から目線・マウンティング表現がない
- [ ] 誤情報・未検証情報がない

### 2. 技術的正確性

- [ ] コードは実際に動作確認済み
- [ ] バージョン情報が明記されている
- [ ] 再現可能な手順になっている

### 3. Front Matter

- [ ] title: 設定済み（60文字以内推奨）
- [ ] emoji: 内容に合った絵文字
- [ ] type: tech または idea
- [ ] topics: 1〜5個の関連タグ
- [ ] published: false → true に変更予定

---

## 推奨チェック（品質向上）

以下は推奨項目です。可能な限り対応してください。

### 4. 記事レビュー

```
/review-article を実行し、総合評価 ⭐⭐⭐⭐☆ 以上であること
```

- [ ] 読者視点: 対象者・メリットが明確
- [ ] 構成: 論理的な流れ、適切な見出し階層
- [ ] 引用: 出典が明記されている
- [ ] SEO: タイトル・topicsが適切

### 5. 校正チェック

```
/proofread を実行し、重大な誤りがないこと
```

- [ ] 誤字脱字がない
- [ ] 表記ゆれがない
- [ ] 文体が統一されている

### 6. AI文章チェック

- [ ] 過剰な箇条書き・リスト化をしていない
- [ ] 定型フレーズ（「〜について解説します」等）がない
- [ ] 曖昧な修飾語（「様々な」「非常に」）を多用していない
- [ ] 具体的なエピソード・失敗談が含まれている
- [ ] 実測値や具体的な数字がある（丸い数字ではない）
- [ ] 自分の感情・意見・好みが表現されている

### 7. プレビュー確認

```
npx zenn-cli preview でプレビュー確認
```

- [ ] レイアウトが崩れていない
- [ ] 画像が正しく表示されている
- [ ] コードブロックのハイライトが正しい
- [ ] リンクが有効である

### 8. 公開日時設定チェック（予約公開の場合）

```
date コマンドで現在日時を確認後、推奨日時を計算
```

- [ ] `date` コマンドで現在日時を確認した
- [ ] 推奨曜日（月〜金）を選んでいる
- [ ] 推奨時刻（月曜12:00、火〜金曜07:00）を設定している
- [ ] 祝日・休日でないことを確認した
- [ ] 同日に他の記事が予約されていない
- [ ] **日付・時刻が正しいことを再確認した**（`published_at`は変更不可のため）

:::message alert
`published_at` は一度設定すると変更できません。設定前に必ず確認してください。
:::

```
┌─────────────────────────────────────┐
│ 1. /check-controversy 実行          │
│    → 🔴 高リスクがあれば修正        │
└────────────────┬────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ 2. /review-article 実行             │
│    → 改善点を修正                   │

                 ↓
┌─────────────────────────────────────┐
│ 3. /proofread 実行                  │
│    → 誤字脱字・表記ゆれを修正       │
└────────────────┬────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ 4. AI文章チェック                   │
│    → AI文章パターンを修正           │
└────────────────┬────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ 5. npx zenn-cli preview             │
│    → 表示確認                       │
└────────────────┬────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ 6. 公開日時の決定                   │

└────────────────┬────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│ 7. published: true に変更           │
│    → published_at を設定（予約時）  │

└─────────────────────────────────────┘
```

## 公開後の確認

- [ ] Zennに正しくデプロイされている
- [ ] OGP画像が正しく生成されている
- [ ] 本番URLでアクセスできる

---

## 緊急時の対応

公開後に問題が発見された場合：

1. **軽微な修正**: 即座に修正してpush
2. **重大な問題**: `published: false` に戻して非公開化
3. **炎上発生時**: 冷静に対応、必要に応じて謝罪・訂正

```bash
# 緊急非公開化
sed -i '' 's/published: true/published: false/' articles/対象記事.md
git add .
git commit -m "緊急非公開化: 〇〇の理由"
git push
```
