# 記事レビュースキル

Zenn記事の品質を多角的にレビューするスキルです。

## 使い方

```
/review-article
```

対象の記事ファイルパスを指定するか、直近で編集中の記事を自動検出します。

## レビュー手順

1. 対象記事ファイルを読み込む
2. 以下の6観点でチェックを実施
3. 問題点と改善提案をレポート出力

---

## A. 読者視点チェック

### チェック項目

- [ ] **対象読者が明確**: 「誰向け」「何が得られるか」が冒頭で分かる
- [ ] **読者メリット**: 読者が得られる具体的な価値がある
- [ ] **前提知識の明示**: 必要な前提知識が書かれている
- [ ] **ゴール設定**: 記事を読み終えた後の状態が想像できる

### 確認ポイント

```
- 「はじめに」セクションに対象者の記載があるか
- 「この記事で学べること」が明示されているか
- 専門用語に説明が付いているか
```

---

## B. 内容の正確性チェック

### チェック項目

- [ ] **技術的正確性**: コード・コマンドが動作する
- [ ] **バージョン明記**: 使用ツール・ライブラリのバージョンが書かれている
- [ ] **再現可能性**: 読者が同じ手順で追試できる
- [ ] **検証済み**: 実際に動かして確認した内容である
- [ ] **環境情報**: OS、言語バージョン等の実行環境が明記されている

### 確認ポイント

```
- コードブロックは実際にコピペして動作するか
- 「〜のはず」「〜だと思う」等の未確認表現がないか
- 手順の抜け漏れがないか（初期設定、依存関係など）
```

---

## C. 引用・出典チェック

### チェック項目

- [ ] **引用元の明記**: 外部情報は出典URLを記載
- [ ] **公式ドキュメント参照**: 可能な限り公式ソースへのリンクがある
- [ ] **ライセンス確認**: 引用コード・画像のライセンスが適切
- [ ] **リンク有効性**: リンク切れがない
- [ ] **引用の正確性**: 原文の意図を正しく伝えているか

### 確認ポイント

```
- 統計データや数値には出典があるか
- 「〜によると」には必ずリンクが付いているか
- 画像・図表の出典が明記されているか
```

---

## D. 構成・読みやすさチェック

### チェック項目

- [ ] **論理的な流れ**: 導入→課題→解決→結果の順
- [ ] **適切な見出し階層**: h2から始まり、階層が乱れていない
- [ ] **段落の長さ**: 長すぎる段落がない（目安: 3-5文）
- [ ] **箇条書きの活用**: 列挙はリスト化されている
- [ ] **コードブロック**: 言語指定・ファイル名がある
- [ ] **視覚的メリハリ**: コード、図、箇条書きで単調さを避けている

### 確認ポイント

```
- 目次を見ただけで内容が把握できるか
- 見出しがh2→h3→h4の順になっているか
- 200文字を超える段落がないか
- コードブロックに言語指定があるか
```

---

## E. SEO・発見性チェック

### チェック項目

- [ ] **タイトル60文字以内**: 検索結果で切れない
- [ ] **topics適切**: 関連性のあるタグ（最大5つ）
- [ ] **emoji適切**: 内容を反映した絵文字
- [ ] **検索キーワード**: 読者が検索しそうなキーワードが含まれている

### 確認ポイント

```
- タイトルに具体的な技術名が含まれているか
- topicsは人気のあるタグを使っているか
- 類似記事と差別化できるタイトルか
```

---

## F. Zenn固有チェック

### チェック項目

- [ ] **Front Matter完備**: title, emoji, type, topics, published がすべて埋まっている
- [ ] **Zenn記法活用**: :::message, :::details等を適切に使用
- [ ] **画像alt**: 画像にはaltテキストがある
- [ ] **埋め込み形式**: @[card], リンクカード等が正しく記述されている

### Zenn記法の例

```markdown
# メッセージボックス
:::message
補足情報
:::

:::message alert
警告・注意
:::

# アコーディオン
:::details 詳細を見る
折りたたまれる内容
:::

# コードブロック
```typescript:example.ts
const code = "ファイル名付き";
```

# 数式
$$
E = mc^2
$$
```
