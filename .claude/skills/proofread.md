# 校正チェックスキル

記事の文章品質を校正・チェックするスキルです。誤字脱字、表記ゆれ、文法エラーなどを検出します。

## 使い方

```
/proofread
```

対象の記事ファイルパスを指定するか、直近で編集中の記事を自動検出します。

## チェック手順

1. 対象記事ファイルを読み込む
2. 4つの観点で校正チェックを実施
3. 修正箇所と修正案をレポート出力

---

## 1. 誤字脱字チェック

### チェック項目

- [ ] 漢字の誤変換がないか
- [ ] ひらがな・カタカナの打ち間違いがないか
- [ ] 脱字（文字の抜け）がないか
- [ ] 衍字（余分な文字）がないか
- [ ] 同音異義語の誤用がないか

### よくある誤字パターン

| 誤 | 正 | 種類 |
|----|----|------|
| 以外と | 意外と | 同音異義語 |
| 一応 | 一様（状況による） | 同音異義語 |
| 的を得た | 的を射た | 慣用句 |
| させて頂く | させていただく | ひらがな |
| ～の様な | ～のような | ひらがな |
| 出来る | できる | ひらがな推奨 |

---

## 2. 表記ゆれチェック

### 文体の統一

- [ ] 「です・ます」調と「だ・である」調が混在していないか
- [ ] 記事全体で統一されているか

```
NG: 「〜です。これは〜である。」（混在）
OK: 「〜です。これは〜です。」（統一）
```

### 技術用語の表記統一

| 統一表記 | 避けるべき表記 |
|----------|----------------|
| GitHub | Github, GITHUB |
| JavaScript | Javascript, javascript |
| TypeScript | Typescript, typescript |
| Node.js | NodeJS, Nodejs, node.js |
| macOS | MacOS, macos, Mac OS |
| VS Code | VSCode, vscode |
| API | api, Api |
| URL | url, Url |
| CLI | cli, Cli |

### 一般用語の表記統一

| 項目 | 統一例 |
|------|--------|
| ユーザ/ユーザー | どちらかに統一 |
| サーバ/サーバー | どちらかに統一 |
| ブラウザ/ブラウザー | どちらかに統一 |
| コンピュータ/コンピューター | どちらかに統一 |

### 数字表記の統一

- [ ] 半角数字で統一されているか
- [ ] 桁区切りのルールが統一されているか

```
NG: 「１０個」「１００円」（全角）
OK: 「10個」「100円」（半角）

NG: 「10000件」と「1,000件」が混在
OK: 「10,000件」「1,000件」で統一
```

---

## 3. 文法チェック

### チェック項目

- [ ] 主語と述語が対応しているか
- [ ] 係り受けが正しいか
- [ ] 二重否定になっていないか
- [ ] 冗長な表現がないか
- [ ] 一文が長すぎないか（目安: 80文字以内）

### よくある文法エラー

```
NG: 「私の考えは、〜と思います」（主語述語不一致）
OK: 「私は〜と思います」または「私の考えは〜です」

NG: 「決して〜ないわけではない」（二重否定）
OK: 「〜である」「〜することもある」

NG: 「することができる」（冗長）
OK: 「できる」「〜する」
```

### 冗長表現の修正例

| 冗長 | 簡潔 |
|------|------|
| することができる | できる |
| 〜という形で | 〜で |
| 〜というもの | 〜 |
| 〜の方が | 〜が |
| まず最初に | まず / 最初に |
| 一番最初 | 最初 |
| 後で後述する | 後述する |
| 約〜ぐらい | 約〜 / 〜ぐらい |

---

## 4. 読点・句読点チェック

### チェック項目

- [ ] 読点（、）の位置が適切か
- [ ] 句点（。）が文末にあるか
- [ ] 箇条書きの末尾の句点が統一されているか

### 読点のルール

```
# 読点を打つべき場所
- 主語の後（長い主語の場合）
- 接続詞の後
- 並列する語句の間
- 誤読を防ぐ場所

NG: 「ここではきものを脱いでください」
OK: 「ここでは、きものを脱いでください」
OK: 「ここで、履き物を脱いでください」
```

### 箇条書きの統一

```
# パターン1: 句点あり
- 項目1です。
- 項目2です。
- 項目3です。

# パターン2: 句点なし
- 項目1
- 項目2
- 項目3

# NG: 混在
- 項目1です。
- 項目2
- 項目3です
```

## チェック出力フォーマット

```markdown
# 校正チェック結果

## 検出件数サマリー
| カテゴリ | 件数 |
|----------|------|
| 誤字脱字 | 2件 |
| 表記ゆれ | 3件 |
| 文法 | 1件 |
| 句読点 | 0件 |

## 修正が必要な箇所

### 誤字脱字
1. 行15: 「以外と」→「意外と」
2. 行42: 「させて頂く」→「させていただく」

### 表記ゆれ
1. 「Github」と「GitHub」が混在
   - 行8: Github → GitHub
   - 行25: Github → GitHub
2. 文体の混在
   - 行30: 「〜である」→「〜です」

### 文法
1. 行55: 「することができます」→「できます」（冗長）

### 句読点
- 問題なし
```

## 自動検出パターン（参考）

以下のパターンは自動検出の対象です：

```
# 誤字脱字パターン
- 「以外と」（→意外と）
- 「一同に」（→一堂に）
- 「延々と」「永遠と」の混同

# 表記ゆれパターン
- 大文字小文字の不統一（github/GitHub）
- 長音の有無（サーバ/サーバー）
- 送り仮名の違い（引き続き/引続き）

# 冗長パターン
- 「〜することができる」
- 「〜という形で」
- 「〜というもの」
```
