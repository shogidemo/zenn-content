# new-article

新しいZenn記事を作成するスキルです。

## 使い方

```
/new-article
/new-article テーマについてのヒント
```

## 手順

1. **テーマの確認**
   - ユーザーがテーマを指定していない場合、以下の候補から選んでもらう：
     - AI導入・活用に関するテーマ
     - 生産性向上に関するテーマ
     - 開発体制改善に関するテーマ
     - その他（ユーザーが自由に指定）

2. **記事の構成を決める**
   - 記事のターゲット読者を確認
   - 記事のゴール（読者が得られること）を確認
   - 大まかな見出し構成を提案

3. **記事ファイルを作成**
   - `npx zenn-cli new:article --slug <slug>` を実行
   - slugは内容を表す英語のケバブケース（例: `introducing-claude-code-to-team`）

4. **Front Matterを設定**
   - title: 記事タイトル（60文字以内）
   - emoji: 内容に合った絵文字を提案
   - type: `tech`（技術記事）を基本
   - topics: 関連するタグを最大5つ提案
   - published: `false`（下書き状態）

5. **テンプレートを適用**
   - `templates/tech-article.md` の構成をベースに
   - 見出し構成を記入

6. **次のステップを案内**
   - 記事ファイルのパスを表示
   - プレビュー方法を案内（`/preview`）
   - 執筆のヒントを提供

## 注意事項

- slugは一度決めたら変更しない（URLが変わってしまう）
- topicsは小文字英数字とハイフンのみ使用可能
- 機密情報は含めないよう注意喚起する
